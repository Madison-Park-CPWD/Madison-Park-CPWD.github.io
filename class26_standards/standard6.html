Standard 6: Fundamentals of UX and UI Design

Students will be able to evaluate user needs, system requirements and design options to produce a high-quality user experience model.

Exposed Competent Proficient Advanced
Students can recall the basic idea of the standards. Student can explain the standard correctly. Student can apply or demonstrate the standard correctly. Student can evaluate the strengths and weakness of the standard and conclude when to use it.

6.a Research and identify the reasons for making software accessible.

6.b Research and identify methods for implementing software accessibility.

6.c Document the user goals, needs, behaviors, and preferences.

6.d Develop personas, research demographics, and identify user types for software.

6.e Develop user experience models that demonstrate an understanding of different audiences.

6.f Research and define where the software will provide feedback on defaults, status, error, validation, and instructions to the user.

6.g Research and identify user research methodologies such as usability tests, user interviews, contextual inquiries, and user observations.

6.h Describe design considerations for different display form factors.

6.i Define different user interface modalities.

6.j Create a simplified interface layout for part of a software application using an appropriate graphic design tool.

6.k Build a storyboard of the user interface for a software application.

6.l Create a prototype from an interface layout and storyboard.
