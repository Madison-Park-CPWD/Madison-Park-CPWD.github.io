Intermediate Python Mini-Certificate

Prerequisites: Completion of Mini-Certificates 1–19 (Foundations through Functions)

Final Goal: Students will write modular, reusable, and well-structured Python code using functional, object-oriented, and file-based programming techniques.

Learning Targets

  • Use lambda functions and built-in functions (map, filter, sorted) for functional programming
  • Apply list comprehensions to create lists concisely and efficiently
  • Perform basic file I/O operations including reading and writing text files
  • Define classes and create objects with __init__ and instance attributes
  • Add methods that use self to manipulate object state
  • Implement inheritance and use super() to extend parent behavior
  • Customize object behavior with special methods like __str__, __repr__, and __eq__
  • Choose between functional and OOP approaches based on problem requirements

Proof

Each checkbox represents a proof of understanding. Students must demonstrate mastery through creation, explanation, or application.

Lambda & Built-in Functions

  • Lambda Expression Mastery – Write 5+ lambda functions (e.g., lambda x: x*2, lambda s: s.upper()) and use them in appropriate contexts
  • Use with map() and filter() – Apply map() and filter() with lambda to transform and filter a list (e.g., double numbers, extract evens)
  • Built-in Function Lab – Use len(), sum(), max(), min(), sorted(), type(), and isinstance() in 10+ real-world data processing scenarios
  • Functional Data Pipeline – Chain map(), filter(), and sorted() to process a list of dictionaries (e.g., student records) and extract filtered, sorted results
  • Other: ________________________________

List Comprehensions

  • Basic List Comprehension – Write list comprehensions to transform lists (e.g., [x**2 for x in range(10)]) in 8+ examples
  • Conditional Filtering – Use if conditions in list comprehensions to filter elements (e.g., [x for x in numbers if x % 2 == 0])
  • Nested Comprehensions – Create nested list comprehensions for 2D data processing (e.g., flattening a matrix)
  • String Processing – Use list comprehensions to process strings (e.g., extract vowels, convert to uppercase)
  • Other: ________________________________

Basic File I/O

  • File Reading – Open and read text files using with open() context manager, processing content line by line
  • File Writing – Create and write to text files, handling both write and append modes safely
  • Error Handling – Implement try/except blocks to handle FileNotFoundError and other I/O exceptions
  • Data Processing Pipeline – Read data from a file, process it (e.g., filter, transform), and write results to a new file
  • Other: ________________________________

Classes & __init__

  • Class Definition Practice – Define 3+ classes (e.g., Student, BankAccount, Product) using class Name: syntax with proper docstrings
  • __init__ Method Implementation – Add a constructor that sets instance attributes (e.g., self.name, self.balance) from parameters
  • Object Creation & Initialization – Instantiate objects with different data and verify attributes are set correctly
  • Attribute Access & Modification – Print and modify instance attributes after creation, demonstrating object state
  • Other: ________________________________

Methods & self

  • Instance Method Creation – Add methods that use self to access or modify object data (e.g., account.deposit(100), student.get_grade())
  • Method Call & Behavior – Call methods on instances and verify correct behavior and state changes
  • self Understanding – Explain why self is required in method definitions and demonstrate that it refers to the instance
  • Method Chaining (Optional) – Design methods that return self to enable chaining (e.g., builder.set_name("Alice").set_age(25))
  • Other: ________________________________

Inheritance & super()

  • Child Class Definition – Create a subclass (e.g., SavingsAccount(BankAccount), Teacher(Person)) that inherits from a parent class
  • Method Overriding – Redefine a parent method in the child class (e.g., calculate_interest(), get_role())
  • super() Usage – Use super().__init__() to extend parent initialization and avoid code duplication
  • Polymorphism Demonstration – Call the same method on different objects (parent and child) and show different behavior
  • Other: ________________________________

Special Methods (__str__, __repr__, __eq__)

  • __str__ Implementation – Add __str__ to return a human-readable string (e.g., "Student: Alice, Grade: A")
  • __repr__ Implementation – Add __repr__ to return a detailed, unambiguous string for debugging (e.g., "Student('Alice', 'A')")
  • Print Object Behavior – Verify that print(obj) uses __str__ and interactive mode uses __repr__
  • __eq__ Custom Comparison – Implement __eq__ to define equality (e.g., two Student objects are equal if names match)
  • Other: ________________________________
✅ Overall Mastery
Demonstrates intermediate-level proficiency in Python by combining functional and object-oriented programming techniques. Creates reusable classes with proper initialization, methods, inheritance, and special methods. Uses lambda, list comprehensions, and built-in functions for clean, expressive code. Performs basic file I/O operations safely and efficiently. Understands when to use OOP vs functional approaches.

Success Indicators

Students are ready for advanced Python (web frameworks, data science, automation) when they can:

  • Design classes that model real-world entities with attributes and behaviors
  • Use inheritance to extend functionality and avoid code duplication
  • Customize object representation and comparison with special methods
  • Apply functional programming tools (lambda, map, filter, list comprehensions) to process data
  • Perform basic file reading and writing operations with proper error handling
  • Choose between procedural, functional, and OOP styles based on problem context
  • Debug object state and method calls using print() and type()
  • Explain the purpose of self, super(), and special methods in their own words

Integration Notes

  • Teach After: Functions (Mini-Cert 17–20) and basic data structures
  • Pair With: Final project work (e.g., building a student management system that reads/writes data to files)
  • Optional Extension: Introduce @property, @staticmethod, or dataclass for advanced learners

This Intermediate Python Mini-Certificate serves as a bridge between foundational syntax and advanced application development, preparing students for real-world programming tasks and deeper study in Python-based frameworks. The addition of list comprehensions and basic file I/O enhances students' ability to process data efficiently and work with persistent storage.
