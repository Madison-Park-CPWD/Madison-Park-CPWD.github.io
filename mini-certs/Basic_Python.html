Basic Python Certificate

Prerequisites: None

Final Goal: Students will write clean, readable Python code using fundamental programming concepts and data types.

Learning Targets

  • Apply the principles of The Zen of Python to code structure and readability
  • Create and manipulate basic data types (strings, integers, floats, booleans)
  • Use variables and understand Python's dynamic typing system
  • Follow PEP 8 style guidelines for clean, readable code
  • Use None and understand identity vs equality (is vs ==)
  • Write conditional logic using if, elif, and else
  • Implement for and while loops for iteration
  • Understand truthiness and falsiness in boolean contexts

Proof

Each checkbox represents a proof of understanding. Students must demonstrate mastery through creation, explanation, or application.

The Zen of Python & Code Philosophy

  • Run and Recite – Execute import this in the Python interpreter and transcribe all 19 principles accurately
  • Explain 8 Principles – Choose 8 lines (e.g., "Simple is better than complex") and write a 2–3 sentence explanation of their meaning in code
  • Code Example for Each – For each of 8 explained principles, write a short code snippet that follows and one that violates the principle
  • Refactor with Zen – Take a poorly written script and refactor it to align with at least 5 Zen principles, annotating changes

Variables & Dynamic Typing

  • Declare & Reassign – Create a variable and reassign it to int, str, list, and bool; explain why this is allowed in Python
  • Type Inspection Lab – Use type() and isinstance() to verify types in 10+ scenarios with literals and variables
  • Naming Practice – Write 10 variables using snake_case, avoiding invalid names (e.g., class, 1var), and explain naming rules
  • Dynamic Behavior Demo – Write a function that accepts any type and prints its type, proving dynamic typing in action

Comments & PEP 8 Style

  • Comment Use – Add meaningful inline and block comments to 5 scripts explaining logic flow
  • PEP 8 Formatting – Format a messy script to follow PEP 8 (indentation, spacing, line length ≤79)
  • Docstring Practice – Write docstrings for 3 functions using triple quotes and describe parameters/return values
  • Style Audit – Use a linter (e.g., pylint or VS Code) to identify and fix 10+ PEP 8 violations in a provided script

Strings

  • String Creation – Create strings using single, double, and triple quotes; explain use cases for each
  • Indexing & Slicing – Access characters and substrings using [ ] and [:] in 8+ examples (positive/negative indices)
  • Method Mastery – Use .upper(), .lower(), .strip(), .replace(), .split() in real tasks (e.g., clean user input)
  • f-String Practice – Use f-strings to insert variables into 6+ dynamic messages (e.g., f"Hello {name}")

Integers & Floats

  • Integer Operations Lab – Perform +, -, *, /, //, %, ** in 10+ calculations with integers and floats
  • Float Precision Workshop – Show a case where 0.1 + 0.2 != 0.3, explain why, and use round() or decimal to fix
  • Number Method Mastery – Use int(), float(), str() to convert between types in 8+ input-handling cases
  • Mathematical Validation – Create functions that verify number inputs and handle edge cases like non-numeric strings or None
  • Other: ________________________________

Booleans & Truthiness

  • Boolean Creation Practice – Generate True/False using comparisons (e.g., 5 > 3) and bool() conversion
  • Truthiness Investigation – Document which values (0, "", [], None, 1, "hello") evaluate to True/False in 12+ examples
  • Logical Operators – Use and, or, not in 6+ compound conditions (e.g., age >= 18 and has_id)
  • Short-Circuit Demo – Write code showing how and/or skip evaluation when possible (e.g., safe_check and risky_operation())

None & Identity

  • Use None Correctly – Assign None to indicate absence of value in 5+ functions (e.g., find_user() returns None if not found)
  • Compare with is – Use is None and is not None instead of == None in 8+ validation checks
  • Identity vs Equality – Explain the difference between is and == using id() and examples (e.g., [] == [] but [] is [] is False)
  • Function Return Analysis – Write a function that returns None implicitly and explain when this happens

Branching (if, elif, else)

✅ Overall Mastery
Demonstrates foundational proficiency in Python by writing clean, readable code using basic data types, variables, and control flow. Applies The Zen of Python principles to code structure, uses proper naming and formatting, and implements conditional logic and iteration effectively.

Success Indicators

Students are ready for intermediate Python when they can:

  • Write clean, properly formatted code following PEP 8 and The Zen of Python
  • Choose the right data type (string, int, float, bool) for the task and explain their choice
  • Control program flow using if, for, and while with proper logic and termination
  • Handle user input safely, validate data, and prevent crashes
  • Explain their code design, variable naming, and logic flow clearly
  • Use type() and isinstance() to check data types before processing
  • Debug systematically using print() and step-by-step reasoning
  • Demonstrate understanding of truthiness, dynamic typing, and object identity (is vs ==)

Integration Notes

  • Teach After: Introduction to programming concepts
  • Pair With: Simple projects like a number guessing game, quiz app, or text-based adventure
  • Assessment: Use code reviews and debugging exercises to evaluate understanding

This Basic Python Certificate provides a solid foundation in Python programming fundamentals, preparing students for intermediate concepts like functions, data structures, and object-oriented programming. The structured progression ensures students build essential skills before advancing to more complex topics.
